# 事件信号监控引擎

本项目是一个本地运行的加密市场事件与量化信号监控系统。系统持续采集市场数据和消息数据，生成纸面告警候选，并通过模拟账户记录持仓、成本、收益和风险指标。

## 主要功能

- 统一高频循环每 1 分钟执行完整监控：行情、资金费率、OI、Polymarket、交易所公告、可选 Whale Alert、内置 RSS、NewsNow 国内外热榜、GDELT 国际新闻、事件复核、模型校准和交易复盘均在同一轮完成。
- 单轮超过 60 秒时不会并发叠加；下一轮会在当前轮结束后再启动，避免重复请求和账户状态竞争。
- 使用统一事件结构接收新闻、热榜、公告、预测市场和链上事件，并进行关键词过滤、热榜排名跟踪、来源分级、近似故事聚类与跨源佐证。
- 在交易所公告入口过滤例行上币、新现货交易对和新永续合约通知；下架、暂停、风险与安全公告仍保留。
- 对 Polymarket 的加密资产价格阈值市场跟踪多方/空方隐含概率、多空比和相较上轮变化，并纳入事件方向与影响评分。
- 每条消息保存事件时间与采集时间，并按来源类型使用不同半衰期计算时效权重；过时消息会衰减影响分但仍可在消息面复核。
- Polymarket 市场一旦进入监控，会按市场 ID 每分钟持续更新双方隐含概率、成交量、流动性和历史观测，直到上游明确标记关闭；关闭后停止请求但保留最终状态。历史采用分层留存：最近 3 小时分钟级、最近 7 天小时级、最长 5 年日级。
- 消息面缺失时继续使用数学模型分析，不中断候选生成。
- 支持保守型和激进型模拟策略风格。
- 开仓胜率硬门槛不是固定百分比：以计入手续费和滑点后的保本胜率为起点，再按历史校准误差、有效样本量、行情状态、波动扩张、消息与数学方向分歧动态升降；保守型和激进型仅保留不同的安全上下限。
- 模拟合约与现货账户，计入手续费、滑点和资金费率。
- 展示净值、回撤、滚动夏普、逐笔盈亏和成本侵蚀。
- 支持按每 X 笔已平仓交易触发批量复盘；新交易保存开仓时因子、权重、模型版本、消息事件和完整决策计算快照。
- 复盘使用扣除手续费、滑点和资金费率后的净 R，展示因子关联、MFE/MAE、持仓路径和候选权重。
- 候选方向权重必须经过按时间顺序切分的影子验证，满足样本和改善门槛后才允许手动或自动晋升，并支持回滚上一版本。
- 数学层包含 GBM、GARCH、HMM、泊松事件到达、贝叶斯后验校准和 Markowitz 配置。
- 12 个 RSS 与 7 个 NewsNow 热榜来源直接内置；不依赖 RSSHub、DailyHotApi、TrendRadar 进程或 Docker，页面仅允许调整保留关键词。
- Whale Alert 已降为可选兼容源，未配置 API Key 不再产生系统告警。

## 安全边界

本项目默认仅执行 `paper-alert-only` 模拟告警，不包含实盘下单逻辑。任何真实交易接入都必须独立实现权限隔离、签名、限额、审计和人工授权。

## 一键打开服务器仪表盘

完成一次性的专用 SSH 密钥授权后，Windows 用户可以双击 `打开服务器前端.cmd`。脚本会把本机
`127.0.0.1:18788` 安全转发到服务器的 `127.0.0.1:8788`，确认接口可用后自动打开浏览器。

- 不会把仪表盘端口暴露到公网。
- 不在脚本中保存服务器密码。
- VPN 切换或本地出口 IP 改变导致连接中断时，后台进程会自动重连。
- 双击 `关闭服务器前端隧道.cmd` 可以停止本地隧道；服务器上的模拟监控不受影响。

API Key 保存在本地 `.runtime/` 或 `.env` 中。这两个路径均被 Git 忽略。不要把真实密钥写入源码、README、Issue 或提交记录。

无证据表明新闻聚合、大模型推理、Polymarket 赔率或当前数学模型能够稳定盈利。历史模拟结果不能证明未来收益。

交易后复盘衡量的是因子与结果之间的预测关联，不是因果证明。功能加入前建立的旧持仓没有开仓时因子快照，平仓后会明确标为不可归因样本，不会使用事后数据伪造决策链路。

## 环境要求

- Windows 10/11
- Node.js 20 或更高版本
- PowerShell 5.1 或更高版本

## 安装

```powershell
npm ci
Copy-Item .env.example .env
```

`.env` 中的 Whale Alert 密钥可以留空。消息来源清单固定在源码中，运行时配置不能覆盖来源；仪表盘“消息面”区域只保存启用状态和保留关键词。内置 RSS 由项目直接请求和解析，热榜直接调用 NewsNow 缓存接口，不需要额外启动 Docker、RSSHub、DailyHotApi、TrendRadar 或 NewsNow 服务。

消息进入评分前会按标题相似度合并为故事簇。重复报道只保留来源等级更高的代表项；等级相同时优先保留本轮响应更快的来源。其余来源只作为交叉佐证，不会重复放大事件分数。热榜来源保存首次出现时间、当前/上一轮排名、最佳排名、出现次数和排名动量。

## 启动

启动统一高频循环：

```powershell
npm run signal:loop
```

启动本地仪表盘：

```powershell
npm run dashboard
```

浏览器访问：`http://127.0.0.1:8788/`

停止统一高频循环：

```powershell
npm run signal:stop
```

## 验证

```powershell
npm run check
npm audit
```

## 数据与状态

运行报告、账户状态、聚合器配置、连接状态、日志和本地凭据均写入：

```text
.runtime/event-signal-monitor/
```

该目录不应提交到 Git。
